# 역할 기반 권한 관리(RBAC) 설계서

## 1. 개요
본 문서는 llm-chat-agent 프로젝트의 사용자 역할(Role)에 따른 기능별 접근 권한 정의를 다룹니다. 시스템은 IDP(Identity Provider)로부터 전달받은 그룹 정보를 바탕으로 권한을 제어합니다.

## 2. 역할 정의
시스템은 크게 두 가지 역할을 정의하며, 세션 생성 시 OIDC의 `groups` 클레임을 통해 이를 확인합니다.

| 역할 (Role) | 설명 | 비고 |
| :--- | :--- | :--- |
| **Admin** | 시스템 관리자. 모든 기능에 대한 풀 액세스 권한을 가짐. | `groups`에 "Admin" 포함 |
| **User** | 일반 사용자. 서비스 이용 및 기본적인 지식 관리 권한을 가짐. | `groups`에 "User" 포함 |

## 3. 기능별 접근 권한 (Matrix)

| 대분류 | 세부 기능 | 엔드포인트 | User | Admin |
| :--- | :--- | :--- | :---: | :---: |
| **UI 접근** | 메인 채팅 화면 | `/` | ✅ | ✅ |
| | RAG 콘솔 화면 | `/rag` | ✅ | ✅ |
| | **관리자 설정 화면** | `/admin` | ❌ | ✅ |
| **채팅** | 실시간 LLM 대화 API | `/chat` | ✅ | ✅ |
| **RAG 관리** | 콜렉션 생성/수정/삭제 | `/api/collections/**` | ✅ | ✅ |
| | 도메인 생성/수정/삭제 | `/api/domains/**` | ✅ | ✅ |
| **지식 데이터** | 개별 지식 등록/수정/삭제 | `/api/rag/knowledge/**` | ✅ | ✅ |
| | 통합 RAG 검색 | `/api/rag/search` | ✅ | ✅ |
| **벌크 작업** | **삭제 대상 건수 확인** | `/api/rag/delete-count` | ❌ | ✅ |
| | **지식 데이터 일괄 삭제** | `/api/rag/bulk-delete` | ❌ | ✅ |
| | **엑셀 일괄 업로드** | `/api/rag/bulk-upload` | ❌ | ✅ |

## 4. 구현 상세

### 4.1 스키마 정의 (UserInfo)
`apps/api/schemas.py`의 `UserInfo` 모델에서 `is_admin` 프로퍼티를 통해 권한 여부를 판단합니다.
```python
class UserInfo(BaseModel):
    sub: str
    preferred_username: str
    groups: List[str]
    
    @property
    def is_admin(self) -> bool:
        return "Admin" in self.groups
```

### 4.2 권한 체크 로직
1. **로그인 필터링**: `auth_callback`에서 "Admin" 또는 "User" 그룹이 없는 경우 세션 생성을 차단합니다.
2. **UI 라우팅 보호**: FastAPI 엔드포인트에서 세션의 `groups`를 확인하여 접근을 제어합니다.
3. **API 레벨 보호**: `get_current_user` 의존성 주입 후, 비즈니스 로직 진입 전 `user.is_admin` 또는 `user.groups`를 체크하여 `403 Forbidden`을 반환합니다.

## 5. 제약 및 주의사항
- 본 권한 체계는 운영 환경의 IDP 설정에 의존합니다.
- 향후 "Viewer" 등 읽기 전용 권한 추가 시, 위 매트릭스의 확장이 필요합니다.
