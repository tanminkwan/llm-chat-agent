# Phase 1 상세 설계서 (Auth & Infrastructure)

## 1. 인증 아키텍처 (OIDC Flow)

### 1.1 컴포넌트 구성
- **IDP Provider**: `mwm-idp` (`https://idp.mwm.local:20443`)
- **OIDC Client**: FastAPI (Authlib 라이브러리 활용)
- **Token Store**: 초기 단계에서는 세션 기반(Starlette SessionMiddleware)으로 관리

### 1.2 인증 시퀀스
1. 사용자가 `/login` 엔드포인트 호출
2. `OAuth.mwm-idp.authorize_redirect()` 수행
3. IDP 인증 후 `/auth/callback`으로 리다이렉트
4. `authorize_access_token()`을 통해 토큰 획득
5. JWT `id_token` 디코딩 및 `groups` 클레임 확인

## 2. 권한 및 보안 설계

### 2.1 토큰 검증 로직 (`libs/core/auth.py`)
- **Algorithm**: `RS256`
- **Public Key**: `https://idp.mwm.local:20443/oauth/jwks`에서 실시간 로드 (향후 캐싱 처리)
- **Claims**: `iss`, `aud`, `exp`, `sub` 검증 필수

### 2.2 FastAPI Security Dependency
- `get_current_user`: Bearer 토큰 검증 후 `UserInfo` 객체 반환
- `admin_required`: `UserInfo.is_admin` 필드 체크

## 3. 서버 및 SSL 설정

### 3.1 Uvicorn 설정
- **Host**: `0.0.0.0`
- **Port**: `21443`
- **SSL Key**: `certs/key.pem`
- **SSL Cert**: `certs/cert.pem`

### 3.2 디렉토리 구조 (Phase 1 기준)
- `libs/core/auth.py`: OIDC 및 JWT 로직
- `libs/core/settings.py`: Pydantic 기반 설정 관리
- `apps/api/main.py`: FastAPI 서버 및 엔드포인트

## 4. 데이터 모델
### 4.1 UserInfo (Value Object)
- `sub`: 유저 고유 식별자
- `username`: 유저 아이디
- `groups`: 리스트 형태의 그룹 정보
- `is_admin`: 그룹 내 'Admin' 존재 여부

## 5. 예외 처리 전략
- `401 Unauthorized`: 토큰 만료, 서명 불일치, 토큰 누락
- `403 Forbidden`: 유효한 사용자이나 필요한 권한(Admin)이 없는 경우
- `500 Internal Server Error`: IDP 서버 통신 불가 등
