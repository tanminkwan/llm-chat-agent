# P07: 대화 쓰레드(Thread) 관리 및 관측성 강화 요구사항

## 1. 개요
- 기존의 불분명한 '세션' 개념을 폐기하고, 화면 단위의 연속된 대화인 '쓰레드(Thread)' 개념을 도입함.
- 모든 API 규격과 로그 필드명을 `thread_id`로 통일하여 명확성을 확보함.

## 2. 목표
- **용어 단일화**: 시스템 전반에서 `session_id`를 `thread_id`로 교체함.
- **기억의 단위 확립**: 화면에 대화가 표시되는 동안을 하나의 쓰레드로 정의하고, 새로고침 시 리셋함.

## 3. 세부 요구사항
### 3.1 API 및 클라이언트 수정
- **필드명 변경**: `/chat` API 요청 바디의 필드명을 `session_id`에서 `thread_id`로 변경함.
- **ID 생성**: 페이지 로드 시마다 새로운 UUID를 생성하여 `thread_id`로 사용함.

### 3.2 백엔드 및 로깅 수정
- **스키마 변경**: `ChatRequest` 모델의 `session_id` 필드를 `thread_id`로 변경함.
- **로그 통합**: `[LLM_LOG]` 접두사가 붙은 모든 로그(요청, 응답, **에러**)에 `thread_id`를 필수로 포함함.

## 4. 제약 사항
- 모든 변경 사항은 기존 테스트 케이스를 통과해야 함. (테스트 코드 내의 필드명도 함께 수정)
