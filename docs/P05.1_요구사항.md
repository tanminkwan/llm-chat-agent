# Phase 05.1 요구사항 정의서 (개인 Prompt 관리 기능)

> 본 단계는 Phase 05 (SPA 전환) 이후 추가되는 기능 단위로, SPA 구조 위에 새 뷰(`/prompts`)를 얹어 개인 시스템 프롬프트의 등록·공유·재사용을 지원한다.

## 1. 개요
사용자가 자주 사용하는 시스템 프롬프트를 매번 직접 작성·복사·붙여넣기 하는 불편을 해소하기 위해, 본인 소유의 시스템 프롬프트를 저장하고 재사용할 수 있는 **개인 Prompt 관리 기능**을 신설한다. 또한 본인 외에 타 user 가 명시적으로 공개한 Prompt 도 조회할 수 있도록 하여 조직 내 노하우 공유를 지원하며, 선택한 Prompt 를 한 번의 클릭으로 Chat 시스템 프롬프트에 적용할 수 있도록 한다.

## 2. 목표
- **개인 Prompt 보관함**: 사용자별로 시스템 프롬프트를 등록·수정·삭제·조회할 수 있는 CRUD UI 제공.
- **선택적 공유**: 등록 시 `타인과 공유` 여부를 선택해 다른 user 가 조회·재사용 가능하도록 함(기본 공유).
- **Chat 시스템 프롬프트 즉시 적용**: 목록 행의 `💬 적용` 버튼을 통해 선택한 Prompt 본문을 Chat 화면의 시스템 프롬프트 입력창에 복사하고 즉시 Chat 화면으로 이동.
- **Chat 화면 ↔ Prompt 화면 양방향 네비게이션**: Chat 상단의 `📝 Prompt 조회` 버튼으로 목록으로 이동, 목록의 `💬 적용` 버튼으로 Chat 으로 복귀.
- **기존 RAG Console 과 동일한 UI 구조·스타일**: 사용자 학습 비용을 최소화하기 위해 RAG Retrieval Console 의 검색 패널 + 테이블 + 사이드바 패턴을 그대로 재사용한다.

## 3. 데이터 모델
신규 테이블 `prompts` 를 추가한다. 테이블 스키마는 다음과 같다.

| 컬럼 | 타입 | 설명 |
| :--- | :--- | :--- |
| `id` | Integer (PK, AutoIncrement) | Prompt 고유 번호 |
| `user_id` | String(255), NOT NULL, Index | 소유자 식별자 (OIDC `sub`) |
| `username` | String(255), Nullable | 소유자 표시 이름 (목록 표시용) |
| `title` | String(200), NOT NULL | Prompt 제목 |
| `content` | Text, NOT NULL | Prompt 본문 (시스템 프롬프트 내용) |
| `is_public` | Boolean, NOT NULL, Default `True` | 타 user 에게 공개 여부 |
| `created_at` | TIMESTAMPTZ, NOT NULL, Default `now()` | 생성일시 |
| `updated_at` | TIMESTAMPTZ, NOT NULL, Default `now()` (onupdate) | 변경일시 |

테이블 생성은 기존 패턴과 동일하게 서비스 기동 시 SQLAlchemy `Base.metadata.create_all` 로 자동 처리된다(별도 마이그레이션 스크립트 없음).

## 4. 권한 요구사항
- **사용 가능 권한**: `User` 그룹(또는 `Admin`).
- **소유자 가드**:
  - 본인 소유(`prompt.user_id == 현재 사용자 sub`) Prompt: 조회/수정/삭제 모두 가능.
  - 타 user 가 등록한 Prompt: `is_public == True` 인 경우에 한해 조회만 가능, 수정/삭제 불가.
  - `💬 Chat 적용` 은 조회 권한이 있으면 누구나 사용 가능 (본인 + 공개 타인).
- **API 보안**: 모든 Prompt API 는 세션/Bearer 인증을 거친 후 위 가드를 적용한다.

## 5. UI 요구사항

### 5.1 사이드바
- 좌측 사이드바에 `📝 Prompt 관리` 네비게이션 링크를 추가한다(Chat / RAG Console 사이).
- 클릭 시 SPA 라우터가 `/prompts` 로 이동하며 페이지 새로고침 없이 Prompt 뷰가 활성화되어야 한다.

### 5.2 Prompt 목록 조회 화면 (`/prompts`)
- **레이아웃**: RAG Retrieval Console 과 동일한 구조 (`search-panel` + `table-container` + `rag-table` 스타일).
- **상단 헤더**:
  - 제목 `Prompt 관리`.
  - 우측에 `+ 신규 Prompt 등록` 버튼 → 등록 사이드바 오픈.
- **조회 조건**:
  - **제목 검색**: 입력한 키워드가 `title` 에 부분 일치(ILIKE `%kw%`).
  - **타인 공유 Prompt 포함 여부**: 체크박스. 비체크 시 본인 것만, 체크 시 본인 + `is_public=True` 인 타 user 의 Prompt 까지 조회.
  - 본인 소유가 아니면서 비공개(`is_public=False`)인 Prompt 는 어떤 조건으로도 조회되지 않는다.
- **목록 테이블 컬럼**:
  | 컬럼 | 비고 |
  | :--- | :--- |
  | 번호 | 행 인덱스 |
  | 제목 | 길면 ellipsis, hover 시 전체 표시 |
  | 소유자 | 본인이면 `나` 배지, 아니면 username 배지 |
  | 공개 | 공개/비공개 배지 |
  | 생성일시 | `YYYY-MM-DD HH:MM` |
  | 변경일시 | `YYYY-MM-DD HH:MM` |
  | Actions | 본인 건: `💬 적용 / View / Edit / Delete` · 타인 건: `💬 적용 / View` |
- **정렬**: `updated_at` 내림차순.

### 5.3 신규 Prompt 등록 / 수정 / 조회 사이드바
- 우측 슬라이드 사이드바 패널을 사용 (RAG 의 `#action-sidebar` 와 동일한 스타일/애니메이션 재사용).
- **공통 입력 항목**:
  - 제목 (필수, 최대 200자)
  - Prompt 내용 (필수, textarea 20행 — 긴 본문 가독성 확보)
  - `타인과 공유` 체크박스 — 신규 등록 시 기본 체크.
- **모드별 동작**:
  - `create`: 모든 필드 입력 가능, `저장하기` 버튼 노출.
  - `edit`: 본인 소유에 한해 가능. 모든 필드 수정 가능, 소유자 정보 표시.
  - `show`: 모든 필드 readOnly, 저장 버튼 숨김, 취소 버튼이 `닫기` 로 변경. 본인/타인 모두 사용.
- **저장 후 동작**: 사이드바 닫고 목록 자동 재조회.

### 5.4 `💬 Chat 에 적용` 액션 (신규)
- Prompt 목록의 모든 행(본인/타인 공유 무관)에 `💬 적용` 버튼을 노출한다.
- 클릭 시 다음 순서로 동작한다:
  1. `GET /api/prompts/{id}` 로 본문을 단건 조회 (권한 가드 통과 시).
  2. Chat 뷰의 `#system-prompt-input` 텍스트영역에 본문을 채워 넣는다.
  3. 시스템 프롬프트 입력창이 접힌 상태였다면 자동으로 펼쳐 사용자가 적용된 본문을 즉시 확인할 수 있게 한다.
  4. SPA 라우터(`pushState`)로 Chat 화면(`/`)을 활성화한다.
- 사용자는 이 한 동작만으로 시스템 프롬프트가 적용된 Chat 화면에 도착하여 곧바로 메시지를 입력할 수 있다.
- 브라우저 뒤로가기를 통해 다시 Prompt 목록 화면으로 복귀할 수 있어야 한다.
- 서버 보안: 본문 조회는 표준 `GET /api/prompts/{id}` 를 그대로 사용해 비공개 타인 건은 403 으로 차단된다.

### 5.5 Chat 화면 시스템 프롬프트 영역 변경
- 기존 `⚙️ 시스템 프롬프트 설정 (클릭하여 열기/닫기)` 영역에 `📝 Prompt 조회` 버튼을 우측 정렬로 추가한다.
- 버튼 클릭 시 SPA 라우터를 통해 `/prompts` Prompt 관리 화면으로 이동한다(브라우저 뒤로가기로 Chat 으로 복귀 가능).
- 기존 시스템 프롬프트 입력 textarea 는 그대로 유지되며, 5.4 의 `💬 적용` 동작에 의해 외부에서 값이 채워질 수 있다.
- Prompt 본문은 길어질 수 있으므로 Prompt 관리 화면에서 CRUD 의 `Edit` / `View` 사이드바를 통해 전체 내용을 확인·수정할 수 있어야 한다.

## 6. API 요구사항
모든 엔드포인트는 `User` 또는 `Admin` 권한을 필요로 한다. URL prefix 는 `/api/prompts`.

| Method | Path | 설명 |
| :---: | :--- | :--- |
| GET | `/api/prompts?include_others={bool}&title={kw}` | 목록 조회 (소유자/공개 필터 + 제목 부분검색) |
| GET | `/api/prompts/{id}` | 단건 조회 (본인 또는 공개 건만 허용) — 5.4 의 적용 동작도 본 엔드포인트를 사용 |
| POST | `/api/prompts` | 신규 등록. body: `{title, content, is_public}` |
| PUT | `/api/prompts/{id}` | 본인 건만 수정. body: `{title, content, is_public}` |
| DELETE | `/api/prompts/{id}` | 본인 건만 삭제 |

응답 모델 `PromptRead` 는 다음 필드를 포함한다: `id, user_id, username, title, content, is_public, is_owner, created_at, updated_at`.
- `is_owner` 는 서버가 현재 요청자 기준으로 계산해 응답(프론트 분기를 단순화).

## 7. 호환성 / 비기능 요구사항
- **백엔드 기존 API 변경 금지**: `/chat`, RAG 관련 엔드포인트는 시그니처 변경 없음.
- **SPA 패턴 유지**: 신규 화면도 기존 4 개 뷰(Chat / RAG / Bulk / Admin) 와 동일한 lazy 초기화 + URL 동기화 규칙을 따른다.
- **JS SRP 유지**: Prompt 관련 클라이언트 로직은 신규 모듈 `prompt.js` 로 분리한다.
- **정적 자원 캐시 무효화**: `index.html` 내 `?v=` 쿼리스트링을 일괄 `v=11` 로 갱신.

## 8. 완료 조건 (Definition of Done)
- [ ] `prompts` 테이블이 서비스 기동 시 자동 생성된다.
- [ ] User 권한 사용자가 `/prompts` 에 접근하여 본인 Prompt 를 등록·수정·삭제·조회할 수 있다.
- [ ] `타인 공유 Prompt 포함` 체크박스로 공개된 타 user Prompt 도 조회/View 할 수 있다.
- [ ] 본인 외 사용자는 타 user 의 Prompt 를 수정·삭제하는 API 를 호출해도 403 을 받는다.
- [ ] 비공개 Prompt 는 어떤 조회 조건으로도 타 user 에게 노출되지 않는다.
- [ ] Chat 화면 상단의 `📝 Prompt 조회` 버튼을 통해 페이지 새로고침 없이 `/prompts` 로 이동한다.
- [ ] 목록 행의 `💬 적용` 버튼을 누르면 Chat 의 시스템 프롬프트 입력창에 본문이 채워지고, 입력창이 펼쳐진 상태로 Chat 화면이 활성화된다.
- [ ] 비공개 타인 Prompt 에 대해 `💬 적용` 시 서버에서 403 으로 차단된다.
- [ ] Prompt 등록/수정/조회 화면의 본문 textarea 가 충분한 높이로 긴 Prompt 도 가독성 있게 표시된다.
- [ ] 기존 Chat / RAG / Bulk / Admin 뷰의 회귀 결함이 없다.
