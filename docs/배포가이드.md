# 리발소 AI 에이전트 운영 환경 배포 가이드

본 문서는 인터넷이 차단된 **폐쇄망 운영 환경(Air-gapped Production)**에 Docker를 사용하여 서비스를 배포하는 방법을 설명합니다.

---

## 1. 준비 단계 (인터넷 가능 환경)

운영 환경으로 가져갈 Docker 이미지와 필수 설정 파일을 준비합니다.

### 1.1 Docker 이미지 빌드 및 추출
개발 서버에서 아래 명령어를 실행하여 이미지를 빌드하고 파일(`.tar`)로 저장합니다.

```bash
# 1. 이미지 빌드
docker build -t llm-agent:latest .

# 2. 이미지 파일 추출
docker save -o llm-agent_latest.tar llm-agent:latest
```

### 1.2 반입 파일 목록 확인
운영 서버로 전송해야 할 필수 파일들입니다.
- `llm-agent_latest.tar`: 애플리케이션 이미지
- `nginx_latest.tar`: Nginx 이미지 (또는 인터넷 환경에서 pull)
- `docker-compose.yml`: 서비스 오케스트레이션 설정
- `nginx.conf`: Nginx 설정 파일 (SSL Termination 및 SSE 최적화)
- `.env`: 환경 변수 설정 파일
- `certs/`: SSL 인증서 폴더 (Nginx에서 사용)
- `init.sql`: 데이터베이스 초기 설정 스크립트 (참조용)

---

## 2. 설치 단계 (운영 환경 - 폐쇄망)

### 2.1 파일 배치
운영 서버의 적절한 경로(예: `/app/llm-chat-agent`)에 준비한 파일들을 배치합니다.

```text
/app/llm-chat-agent/
├── llm-agent_latest.tar
├── nginx_latest.tar
├── docker-compose.yml
├── nginx.conf
├── .env
└── certs/
    ├── cert.pem
    └── key.pem
```

### 2.2 Docker 이미지 로드
반입된 이미지 파일들을 Docker 시스템에 등록합니다.

```bash
docker load -i llm-agent_latest.tar
docker load -i nginx_latest.tar
```

### 2.3 데이터베이스 준비 (PostgreSQL)
서비스 기동 전, 데이터를 저장할 데이터베이스와 계정을 수동으로 생성해야 합니다. (테이블은 서비스 기동 시 자동 생성됨)

#### [방법 1] Docker로 설치된 DB를 사용하는 경우
애플리케이션과 함께 배포된 Docker 컨테이너를 사용할 때의 접속 방법입니다.
1. **DB 컨테이너 접속**: 터미널에서 아래 명령어를 실행하여 PostgreSQL CLI에 접속합니다.
   ```bash
   docker exec -it postgres psql -U postgres
   ```
2. **SQL 실행 (순서대로 입력)**:
   ```sql
   -- 데이터베이스 생성
   CREATE DATABASE llm_agent;
   
   -- 사용자 생성 및 패스워드 설정 (.env의 DB_PASSWORD와 일치해야 함)
   CREATE USER admin WITH PASSWORD 'password';
   
   -- 모든 권한 부여
   GRANT ALL PRIVILEGES ON DATABASE llm_agent TO admin;
   
   -- 종료
   \q
   ```

#### [방법 2] 외부 PostgreSQL 서버를 사용하는 경우 (Production 권장)
별도의 전용 DB 서버를 사용할 때의 접속 방법입니다.
1. **서버 접속**:
   ```bash
   psql -h <DB_HOST_IP> -U postgres
   ```
2. **데이터베이스 및 유저 생성**:
   ```sql
   CREATE DATABASE llm_agent;
   CREATE USER admin WITH PASSWORD 'your_secure_password';
   GRANT ALL PRIVILEGES ON DATABASE llm_agent TO admin;
   
   -- (PostgreSQL 15 이상 버전인 경우 필수) public 스키마 권한 추가
   \c llm_agent
   GRANT ALL ON SCHEMA public TO admin;
   \q
   ```

### 2.4 Vector DB 준비 (Qdrant)
Qdrant는 지식 데이터를 벡터 형태로 저장합니다. 별도의 스크립트 실행은 필요 없으나, 설정값 확인이 중요합니다.

1. **임베딩 차원(Dimension) 설정**: `.env` 파일의 `EMBEDDING_DIM` 값이 사용하려는 임베딩 모델의 출력 차원과 일치해야 합니다.
   - 예: OpenAI `text-embedding-3-small` -> `1536`
   - 예: HuggingFace 모델 등 -> `768` 또는 `1024` 등
2. **데이터 영속성**: `docker-compose.yml`에서 `./qdrant_data` 폴더가 정상적으로 매핑되어 있는지 확인합니다.
3. **초기화**: 서비스 기동 후 관리자 화면(`📚 Collection`)에서 콜렉션을 생성하면, 시스템이 자동으로 Qdrant 내에 콜렉션 생성 및 필터링용 인덱스(`domain_id`, `created_at`)를 구축합니다.

### 2.5 환경 변수 설정 (.env)
운영 환경에 맞게 `.env` 파일을 수정합니다. **폐쇄망에서는 내부 LLM 게이트웨이 주소가 중요합니다.**

```bash
# 주요 설정 항목
CHAT_LLM_BASE_URL=http://internal-llm-gateway:8000/v1
CHAT_LLM_API_KEY=EMPTY  # 내부 서버가 키를 요구하지 않아도 EMPTY 입력 권장
OIDC_ISSUER=https://internal-idp.mwm.local:20443
```

## 3. 서비스 실행

Docker Compose를 사용하여 서비스를 백그라운드에서 실행합니다.

```bash
# 서비스 기동
docker compose up -d

# 실행 상태 확인
docker ps
```

---

## 4. 사후 검증 및 관리

### 4.1 서비스 접속 확인
브라우저에서 설정된 도메인으로 접속하여 정상 작동 여부를 확인합니다.
- **주소**: `https://llm-agent.mwm.local:21443/`
- **주의**: 보안 정책상 포트 80(HTTP)은 사용하지 않으며, 21443(HTTPS)을 통해서만 서비스에 접속할 수 있습니다.

### 4.2 로그 모니터링
문제가 발생할 경우 아래 명령어로 로그를 확인합니다.

```bash
# Nginx 로그 (인증 및 리버스 프록시 이슈)
docker logs -f nginx

# App 로그 (LLM 로직 및 인증 이슈)
docker logs -f llm-agent
```

### 4.3 서비스 중지
```bash
docker compose down
```

---

## 5. 주의 사항
- **보안 정책**: 포트 80(HTTP)은 호스트로 매핑되지 않습니다. 반드시 **21443(HTTPS)** 포트를 사용하십시오.
- **Nginx 최적화**: `nginx.conf`에 스트리밍 대화를 위한 `proxy_buffering off` 설정이 포함되어 있습니다. 임의로 변경하지 마십시오.
- **인증서**: `certs/` 폴더 내의 인증서 파일명이 `key.pem`, `cert.pem`으로 일치해야 Nginx가 정상 기동됩니다.
- **Docker Volume 강제 마운트 없음**: 소스코드 수동 수정을 지양하고, 변경 사항 발생 시 반드시 이미지를 다시 빌드하여 배포하십시오.
