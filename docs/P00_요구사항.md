# 요구사항 정의서 (Requirement Specification)

## 1. 프로젝트 개요
본 프로젝트는 사용자 질문에 대해 AI 에이전트가 응답하고, RAG(Retrieval-Augmented Generation) 기능을 테스트할 수 있는 AI 에이전트 서비스입니다.

## 2. 시스템 아키텍처 및 환경
- **인프라**: Docker Compose 기반 컨테이너화
- **사용자 인증**: 별도의 User 관리 없이 IDP Client(OIDC)를 통한 인증 대행
- **권한 체계**: Admin, User 두 단계로 구분
- **LLM 구성**:
  - Production: Qwen3-Coder 30B, GPT-OSS 130B 수준의 로컬 모델 (vLLM)
  - Test/Dev: OpenAI(gpt-4o, o4-mini) 또는 Gemini 등가 모델
- **Vector Store**: Qdrant 사용

## 3. 상세 기능 요구사항

### 3.1 Chat UI 및 에이전트
- **대화 기능**: 사용자의 질문에 AI 에이전트가 응답하는 인터페이스 제공
- **동작 제어**: System Prompt를 통해 에이전트의 행동(Behavior)을 개인별로 설정 가능
- **모델 선택**: Task에 따라 `chat-llm` 또는 `reasoning-llm`을 선택하여 응답 생성
- **메모리**: 대화 맥락 유지를 위한 Memory 기능 구현

### 3.2 RAG TEST 기능
- **텍스트 분석**: UI에서 두 텍스트 간의 거리(유사도) 계산 (Qdrant 임베딩 모델 활용)
- **데이터 관리**: 텍스트와 해당 임베딩 벡터를 Qdrant에 저장
- **검색 기능**: 저장된 텍스트들과의 거리 비교를 통해 관련 컨텐츠를 찾아주는 검색 기능 구현
