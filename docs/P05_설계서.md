# Phase 05 설계서 (Single Page Application 전환)

## 1. 아키텍처 개요

### 1.1 변경 대조 (AS-IS vs TO-BE)

| 구분 | AS-IS (다중 페이지) | TO-BE (SPA) |
| :--- | :--- | :--- |
| **HTML 파일** | `index.html` / `rag.html` / `bulk.html` / `admin.html` 4 개 | **`index.html` 1 개 (셸)** |
| **페이지 전환 방식** | 사이드바 클릭 → 서버에 새 HTML 요청 → 전체 리로드 | 사이드바 클릭 → `pushState` + 뷰 토글 (서버 요청 없음) |
| **상태 보존** | 전환 시 **전부 초기화** (DOM 파괴) | **전부 유지** (모든 뷰가 DOM 에 상주) |
| **서버 라우트** | UI 라우트마다 다른 HTML 반환 | 모든 UI 라우트가 동일한 SPA 셸 반환 |
| **클라이언트 라우터** | 없음 | `app.js` 의 History API 기반 경량 라우터 |

## 2. 세부 설계

### 2.1 백엔드 라우트 통합 (`apps/api/main.py`)

모든 UI 라우트가 동일한 `apps/api/static/index.html` 을 반환한다. 인증 체크는 헬퍼 함수로 일원화한다.

```python
SPA_INDEX_PATH = os.path.join(static_dir, "index.html")

def _require_user(request, admin_only=False):
    user = request.session.get('user')
    if not user:
        return RedirectResponse(url="/auth/login")
    groups = user.get('groups', [])
    if admin_only:
        if "Admin" not in groups:
            return RedirectResponse(url="/static/unauthorized.html")
    else:
        if not any(r in groups for r in ["Admin", "User"]):
            return RedirectResponse(url="/static/unauthorized.html")
    return None

@app.get("/")     # Chat
@app.get("/rag")  # RAG Console
@app.get("/bulk") # Batch Upload
@app.get("/admin", admin_only=True)  # Admin Dashboard
```

- `/admin` 만 `admin_only=True` 로 그룹 체크.
- 기존에 흩어져 있던 인증 분기를 헬퍼로 통합해 라우트 본문을 단순화.
- 기존 `rag.html`, `bulk.html`, `admin.html` 은 더 이상 서버 라우트에서 사용되지 않음(static 마운트는 유지되지만 수명 종료).

### 2.2 SPA 셸 (`apps/api/static/index.html`)

- `<body data-view="chat">` 으로 현재 활성 뷰를 표기. CSS 가 이 속성을 읽어 사이드바 영역을 분기 노출.
- `<main id="view-container">` 안에 4 개의 `<section class="view" id="view-{name}">` 배치. 활성 뷰만 `.active` 클래스 부여.
- 사이드바 nav-link 는 `data-view` / `data-tab` 속성으로 라우팅 메타 정보를 표시:
  ```html
  <a href="/"     class="nav-link" data-view="chat">💬 Chat</a>
  <a href="/rag"  class="nav-link" data-view="rag">📚 RAG Console</a>
  <a href="/bulk" class="nav-link" data-view="bulk">📤 Batch Upload</a>
  <a href="/docs" target="_blank" class="nav-link">📘 API Docs</a>
  <a href="/admin?tab=collection" class="nav-link"
     data-view="admin" data-tab="collection">📚 Collection</a>
  ```
- `#action-sidebar` (RAG 신규/수정), `#progress-overlay` (Bulk 진행률) 등 `position: fixed` 컴포넌트는 `<main>` 외부 body 직속에 배치하여 활성 뷰와 무관하게 호출 가능.

### 2.3 뷰 활성화 규칙 (CSS)

```css
.view              { display: none; flex: 1; min-height: 0; }
.view.active       { display: flex; }

#view-chat         { flex-direction: column; position: relative; }
#view-rag,
#view-bulk,
#view-admin        { flex-direction: column; padding: 30px; overflow-y: auto; }

/* Chat 전용 사이드바 영역(Temperature 등)은 다른 뷰에서 자동 숨김 */
body:not([data-view="chat"]) .chat-only { display: none !important; }
```

- 뷰별 레이아웃(가로 패딩, 스크롤, 갭)을 `.view` 자식 ID 로 분리.
- `body[data-view]` 속성과 `.chat-only` 클래스를 조합한 컨디셔널 노출.

### 2.4 클라이언트 라우터 (`apps/api/static/js/app.js`)

```javascript
const VIEW_IDS = ['chat', 'rag', 'bulk', 'admin'];
const viewInitialized = { chat: false, rag: false, bulk: false, admin: false };

window.addEventListener('DOMContentLoaded', () => {
    checkLogin();
    loadConfig();
    setupNavigation();

    const { view, tab } = parseLocation();
    showView(view, { tab, push: false });
});

window.addEventListener('popstate', () => {
    const { view, tab } = parseLocation();
    showView(view, { tab, push: false });
});

function parseLocation() {
    const path = window.location.pathname;
    let view = 'chat';
    if (path.startsWith('/rag'))   view = 'rag';
    else if (path.startsWith('/bulk'))  view = 'bulk';
    else if (path.startsWith('/admin')) view = 'admin';
    const tab = new URLSearchParams(window.location.search).get('tab');
    return { view, tab };
}

function showView(view, { tab, push = false, url } = {}) {
    VIEW_IDS.forEach(id => {
        document.getElementById(`view-${id}`)
                ?.classList.toggle('active', id === view);
    });
    document.body.dataset.view = view;
    // 사이드바 nav-link active 토글, URL pushState, 뷰별 lazy init...
}
```

라우터 책임:
1. **초기 라우팅**: 로드 시 URL 을 파싱해 알맞은 뷰를 활성화.
2. **사용자 클릭 처리**: `a.nav-link[data-view]` 클릭을 가로채 뷰 토글 + `pushState`.
3. **뒤로/앞으로 가기 처리**: `popstate` 에서 URL 재해석.
4. **뷰별 lazy 초기화**: 첫 진입 시에만 `initChat() / initRag() / initBulk() / initAdmin()` 호출.
5. **수식자 키 보존**: `Ctrl/Cmd/Shift/Alt + Click` 또는 `target="_blank"` 인 링크는 가로채지 않음.

### 2.5 뷰별 모듈 변경 사항

| 모듈 | 변경 내용 |
| :--- | :--- |
| `chat.js` | 자동 부착되던 `DOMContentLoaded` 핸들러 제거. `initChat()` 함수 신설(텍스트영역 자동 높이/엔터 키 핸들러 부착). `threadId` 는 SPA 세션 동안 유지. |
| `rag.js` | `initRag()` 신설(드롭다운 로드 + 검색창 엔터 키 바인딩). 검색 결과 행의 인라인 onclick 핸들러는 그대로 유지. |
| `bulk.js` | 전역 함수 `loadDropdowns` → **`loadBulkDropdowns`** 로 개명(rag.js 와의 충돌 해소). 자동 `DOMContentLoaded` 호출 제거, `initBulk()` 신설. |
| `admin.js` | 더 이상 사용하지 않는 모달 로더(`openAdminModal` / partial fetch) 제거. `switchTab(tab, {push})` 신설(URL 동기화 포함). `initAdmin()` 신설. |
| `app.js` | 신규 라우터 코드 + 기존 `checkLogin` / `loadConfig` 흡수. |

### 2.6 인라인 핸들러 호환성

기존 HTML 의 `onclick="sendMessage()"`, `onsubmit="saveKnowledge(event)"` 등 인라인 핸들러는 그대로 유지한다. 모든 핸들러는 전역 함수로 정의되어 있어 SPA 통합 후에도 정상 호출된다. 인라인 핸들러는 다음과 같이 검증:

| 핸들러 | 정의 위치 |
| :--- | :--- |
| `sendMessage`, `toggleSystemPrompt` | `chat.js` |
| `performSearch`, `openSidebar`, `closeSidebar`, `saveKnowledge`, `deletePoint`, `openEditSidebar` | `rag.js` |
| `startUpload`, `confirmDelete`, `closeProgress`, `downloadErrors` | `bulk.js` |
| `switchTab`, `saveCollection`, `resetColForm`, `editCollection`, `deleteCollection`, `saveDomain`, `resetDomForm`, `editDomain`, `deleteDomain` | `admin.js` |

### 2.7 ID 충돌 방지

원래 4 개 페이지가 각자 사용하던 동일한 ID(예: `#main-content`, `#admin-modal`, `#nav-collection`, `#nav-domain`)는 SPA 통합 시 충돌을 일으킨다. 이를 다음과 같이 해소한다.

- `#main-content` → 뷰별 ID(`#view-rag`, `#view-bulk`, `#view-admin`)로 분리.
- 사이드바의 admin 하위 메뉴 ID(`#nav-collection`, `#nav-domain`)는 제거하고 `data-view="admin" data-tab="..."` 속성만 사용.
- 이전 admin 모달(`#admin-modal`, `partials/admin-modal.html`)은 사용처가 없어 제거.

## 3. 데이터 흐름

```
사용자 클릭 (e.g. 사이드바 "📚 RAG Console")
  └─ click 이벤트 → app.js 의 라우터가 가로채기
       1) e.preventDefault()
       2) showView('rag', { push: true, url: '/rag' })
            ├─ #view-rag 에 .active 클래스 부여
            ├─ body.dataset.view = 'rag'
            ├─ history.pushState({}, '', '/rag')
            └─ initRag() (최초 1회만)
                  └─ loadDropdowns() (Collection / Domain 목록 로드)
```

## 4. 예외 처리 및 고려 사항

- **인증 만료**: SPA 내부에서 `/api/*` 호출이 401 을 반환하더라도 자동 로그인 리다이렉트는 수행하지 않음. 기존 로직(`alert('로그인이 필요합니다.')`)을 유지.
- **`/admin` 직접 접근 차단**: 비-Admin 사용자가 `/admin` 으로 직접 진입할 경우 서버에서 unauthorized 페이지로 리다이렉트. SPA 내부에서는 admin 사이드바가 노출되지 않으므로 일반 사용자가 admin 뷰를 볼 수 없음.
- **모든 뷰의 동시 마운트**: 비활성 뷰도 DOM 에 존재하므로 초기 페이지 로드 비용이 다소 증가함. 단, 뷰별 데이터 fetch 는 lazy 로 수행하여 네트워크 비용은 최소화.
- **캐시 무효화**: 정적 리소스 URL 에 `?v=9` 쿼리스트링을 부여하여 브라우저 캐시 무효화.

## 5. 테스트 계획

| 항목 | 검증 방법 |
| :--- | :--- |
| 채팅 상태 보존 | Chat → RAG → Chat 이동 후 메시지 내역, 입력값 유지 확인 |

| URL 동기화 | 뷰 전환 시 브라우저 주소창이 알맞게 업데이트, 새로고침 시 동일 뷰 복원 |
| 뒤로/앞으로 | 브라우저 ◀ / ▶ 클릭 시 SPA 가 동일 흐름으로 뷰 복원 |
