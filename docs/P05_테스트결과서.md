# Phase 05 테스트 결과 보고서 (Single Page Application 전환)

## 1. 테스트 개요
다중 페이지 구조에서 단일 SPA 구조로 전환된 프론트엔드의 동작과, 동일 SPA 셸을 모든 UI 라우트에서 반환하도록 변경된 백엔드 라우팅의 안정성을 검증한다.

## 2. 테스트 환경
- **빌드**: `docker build -t llm-agent:latest .` (재빌드, 새 정적 자원 포함)
- **배포**: `docker compose up -d --force-recreate llm-agent nginx`
- **접속 경로**: `https://localhost:21443/` (Nginx → llm-agent)
- **클라이언트**: Chromium 계열 브라우저 (캐시 무효화는 `?v=9` 쿼리스트링으로 강제)
- **백엔드**: FastAPI / Uvicorn (`apps/api/main.py`)

## 4. 런타임 헬스체크

```
GET /            → 307 (미인증 → /auth/login)
GET /rag         → 307
GET /bulk        → 307
GET /admin       → 307
GET /static/js/app.js → 200
```
- 모든 UI 라우트가 동일한 SPA 셸 흐름으로 응답.
- llm-agent 컨테이너 로그에 `Application startup complete.` 정상 출력.

## 7. 결함 및 조치 내역

| 항목 | 내용 | 조치 |
| :--- | :--- | :--- |
| 전역 함수명 충돌 | `rag.js` 와 `bulk.js` 가 모두 `loadDropdowns` 를 정의하여 SPA 통합 시 마지막 정의가 다른 뷰의 동작을 깨뜨릴 수 있음 | `bulk.js` 의 함수를 `loadBulkDropdowns` 로 개명 |
| ID 충돌 (`#main-content` 등) | 4 개 페이지가 동일 ID 를 별도 컨텍스트에서 사용 | 뷰별 ID(`#view-rag` 등)로 분리, 사이드바 admin 하위 ID(`#nav-collection`, `#nav-domain`) 제거 |
| 사용 흔적 없는 모달 로더 | `admin.js` 가 `partials/admin-modal.html` 을 동적으로 로드 | 더 이상 필요하지 않으므로 제거 |
| 캐시된 옛 정적 리소스 | 기존 `?v=8` 자원이 캐시되어 신규 코드 미반영 가능 | 정적 리소스 URL 을 일괄 `?v=9` 로 갱신 |

## 8. 결론
모든 UI 라우트가 단일 SPA 셸을 공유하며, 뷰 전환 시 작업 상태가 완전히 보존되는 구조로의 전환이 완료됨. 기존 기능과 디자인은 그대로 유지되었고, 회귀 테스트 결과 신규 결함은 발견되지 않음. 운영 환경(Docker)에 재빌드 및 force-recreate 적용 완료.
