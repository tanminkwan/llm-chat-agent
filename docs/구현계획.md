# 프로젝트 구현 계획 (Implementation Plan)

## 1. 개요
본 프로젝트는 LangChain과 Qdrant를 활용한 AI 에이전트 서비스로, Chat UI와 RAG 기능을 제공합니다. `CLAUDE.md`의 원칙을 준수하며 단계별로 진행합니다.

## 2. 현재 상태 (Checkpoint)

- [x] 프로젝트 초기화 (uv workspace, Python 3.12)
- [x] 기본 설정 및 LLM Gateway 기본 구조 (구현됨 - 설계서 작성 시 재검토)

## 3. 단계별 추진 계획

### Phase 1: 설계 및 기초 환경 (현재 단계)
1. **상세 설계서 작성 (`docs/설계서.md`)**:
   - 시스템 아키텍처 (FastAPI, LangChain, Qdrant 연동)
   - **인증 아키텍처 (OIDC Flow, JWT 관리, Role Mapping)**
   - 모듈간 인터페이스 (LLM Gateway, Memory Manager, Vector Store)
2. **IDP Client 등록 및 환경 설정**:
   - 외부 IDP 서버에 클라이언트 등록 및 `Client ID/Secret` 확보
   - `.env` 및 `sample.env`에 OIDC 설정 항목 추가 및 현행화

### Phase 2: 코어 모듈 개발 (TDD 기반)
1. **인증 및 권한 모듈 (Auth Module)**:
   - OIDC 클라이언트 구현 (Authlib 등 활용)
   - JWT 검증 및 세션 관리 로직
   - Admin/User 권한 필터(Dependency Injection) 구현
2. **LLM Gateway 고도화**:
   - OpenAI 및 vLLM 지원을 위한 LangChain 기반 Gateway
3. **Memory Manager 구현**:
   - 세션별 대화 이력 관리 및 권한별 시스템 프롬프트 주입
4. **Vector Store (Qdrant) 연동**:
   - 텍스트 임베딩 저장 및 유사도 검색 API

### Phase 3: API 서비스 및 UI
1. **FastAPI 백엔드 구축**:
   - `/chat`: 스트리밍 지원 대화 API
   - `/rag/test`: 거리 계산 및 검색 테스트 API
   - `/auth`: IDP(OIDC) 연동 (MOCK 또는 실제 연동)
2. **Frontend (Chat UI)**:
   - Vanilla HTML/JS 기반의 경량 UI (FastAPI Static file 서비스)

### Phase 4: 테스트 및 문서화
1. **종합 테스트**:
   - 유닛 테스트 커버리지 95% 달성
   - 통합 테스트 (Docker Compose 기반)
2. **최종 결과서 작성**:
   - `docs/테스트결과서.md`
