# Phase 4 상세 설계서 (RAG 통합 관리 콘솔)

## 1. 아키텍처 설계

### 1.1 데이터 흐름
```
[Frontend (rag.html)] 
      ↓ (Filter: col_id, dom_id, query)
[FastAPI (main.py)]
      ↓ (RAGService)
[Qdrant (Vector DB)] -- (Filter by payload.domain_id)
```

## 2. 데이터베이스 및 스토리지 설계

### 2.1 Postgres (Master Data)
- **collections**: RAG 유형 관리
    - `name`: **Primary Key (String)**, Qdrant 콜렉션 명, **수정 불가**
    - `description`: 설명 (Text)
    - `snippet_size_limit`: 스니펫 제한 (Integer)
    - `search_method`: 검색 방식 (String)
- **domains**: 지식 분야 관리 (id, name)

### 2.2 Qdrant (Vector Data)
- **Payload Structure**:
    - `content`: 문서 텍스트
    - `domain_id`: 지식 분야 ID (Integer)
    - `filename`: 출처 파일명
- **Indexing Strategy**:
    - 콜렉션 생성 시 `domain_id` 필드에 대해 **Keyword Index** 적용.
    - `client.create_payload_index(collection_name, "domain_id", "keyword")`

## 3. API 상세 설계

### 3.1 RAG 통합 검색 API
- **Endpoint**: `GET /api/rag/search`
- **Request Parameters**:
    - `collection_id` (int, required)
    - `domain_id` (int, optional)
    - `query` (str, optional)
- **Response**: List of matching chunks with metadata.

## 4. 프론트엔드 UI 설계

### 4.1 화면 레이아웃
- **Search Panel (Top)**:
    - Collection 드롭다운 (Type 선택)
    - Domain 드롭다운 (Field 선택)
    - Text Input (검색어)
    - Search 버튼
- **Data Table (Center)**:
    - 검색 결과 또는 전체 리스트 표시.
    - 컬럼: [유형 | 분야 | 내용 요약 | 출처 | 작업(수정/삭제)]
- **Action Sidebar (Right/Modal)**:
    - 신규 등록 및 수정을 위한 폼 슬라이드.
