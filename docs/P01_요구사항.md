# Phase 1 요구사항 정의서 (Auth & Infrastructure)

## 1. 개요
본 단계에서는 서비스의 보안 기반이 되는 OIDC 인증 시스템을 구축하고, HTTPS 통신 환경을 설정합니다.

## 2. 목표
- `mwm-idp`와 OIDC Authorization Code Flow 연동 완료
- `Admin`, `User` 권한 구분을 위한 토큰 분석 및 검증 로직 구현
- 포트 `21443`에서 작동하는 HTTPS API 서버 구축

## 3. 상세 요구사항

### 3.1 인증 및 권한 (OIDC)
- **Login**: `/login` 호출 시 IDP 로그인 페이지로 정상 리다이렉트되어야 함.
- **Callback**: 인증 후 `/auth/callback`을 통해 토큰(`Access`, `ID Token`)을 획득해야 함.
- **Token Verification**: RS256 알고리즘 및 JWKS를 사용하여 발급된 토큰의 위변조를 검증해야 함.
- **Role Mapping**: ID Token의 `groups` 클레임에 `Admin` 또는 `User`가 포함되어 있는지 확인하여 권한을 부여해야 함.

### 3.2 인프라 및 보안 (SSL)
- **Domain**: `llm-agent.mwm.local` 도메인 사용.
- **Port**: `21443` 포트 고정.
- **SSL**: 로컬 개발용 Self-signed 인증서를 적용하여 브라우저에서 `https`로 접근 가능해야 함.
- **Environment**: `.env` 파일에 모든 비밀 정보가 관리되어야 하며, `Settings` 객체를 통해 안전하게 접근해야 함.

### 3.3 기초 API 구조
- **Health Check**: `/` 경로 접근 시 서버 상태를 반환해야 함.
- **Auth Guard**: 보호된 API 접근 시 유효한 Bearer 토큰이 없으면 `401 Unauthorized`를 반환해야 함.

## 4. 완료 조건 (Definition of Done)
- [ ] `https://llm-agent.mwm.local:21443/login`을 통해 로그인이 성공함.
- [ ] 토큰 검증 로직에 대한 유닛 테스트 통과.
- [ ] 유효한 권한이 없는 사용자가 관리자 전용 API 접근 시 `403 Forbidden` 발생 확인.
