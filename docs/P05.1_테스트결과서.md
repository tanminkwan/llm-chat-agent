# Phase 05.1 테스트 결과 보고서 (개인 Prompt 관리 기능)

## 1. 테스트 개요
신규 추가된 `prompts` 테이블 / `PromptService` / `/api/prompts` 엔드포인트와, SPA 에 통합된 Prompt 관리 화면(검색·CRUD·Chat 연동·`💬 적용`)의 정상 동작과 권한 격리를 검증한다.

## 2. 테스트 환경
- **빌드**: `docker build -t llm-agent:latest .` (Phase 05.1 변경 사항 포함 재빌드)
- **배포**: `docker compose up -d --force-recreate llm-agent nginx`
- **접속 경로**: `https://localhost:21443/`
- **클라이언트**: Chromium 계열 브라우저 (캐시 무효화는 `?v=11` 쿼리스트링으로 강제)
- **계정**: User 권한 계정 2개 (A / B), Admin 1개

### 3.2 신규 테스트 시나리오 요약

#### Repository (`test_prompt_repository.py`)
- 본인만 조회: WHERE 절에 `user_id` 만 있고 `is_public` / OR 결합 없음
- `include_others=True`: WHERE 절에 `user_id OR is_public` 결합
- 제목 키워드: `ILIKE '%kw%'` 가 SQL 에 포함됨
- 정렬: `ORDER BY updated_at DESC`
- BaseRepository 위임: `get_by_id / create / update / delete`

#### Service (`test_prompt_service.py`)
- `get_prompt`: 본인 OK / 공개 타인 OK / 비공개 타인 → `PermissionError` / 미존재 → `ValueError`
- `update_prompt / delete_prompt`: 본인만 OK, 그 외 `PermissionError`
- `create_prompt`: 모든 인자가 repo 에 그대로 전달됨

#### API (`test_api_prompts.py`)
- 목록: 본인만 / `include_others` 포함 / 제목 키워드 / 비-User 그룹 403
- 단건: 본인 200 / 공개 타인 200 (`is_owner=false`) / 비공개 타인 403 / 미존재 404
- 등록: 정상 200 (서버가 username/`is_owner=true` 부여) / 필수 필드 누락 422
- 수정·삭제: 본인 200 / 타인 403 / 미존재 404

```

      "ix_prompts_user_id" btree (user_id)
  ```

## 8. 결함 및 조치 내역

| 항목 | 내용 | 조치 |
| :--- | :--- | :--- |
| 사이드바 클래스 충돌 위험 | RAG 사이드바가 `#action-sidebar` ID 셀렉터로만 스타일링되어 있어 신규 Prompt 사이드바에 동일 외형을 적용하기 어려움 | `.action-sidebar-panel` 클래스를 도입해 `#action-sidebar, .action-sidebar-panel` 양쪽에서 공유. 기존 RAG 사이드바 동작은 변경 없음 |
| 캐시된 옛 정적 리소스 | `?v=10` 자원이 캐시되어 신규 `💬 적용` 동작이 미반영될 가능성 | `index.html` 의 모든 정적 리소스 URL 을 `?v=11` 으로 갱신 |
| `is_owner` 클라이언트 계산 부담 | 프론트에서 현재 사용자 sub 와 매번 비교해야 함 | 서버 응답에 `is_owner` 필드 포함하여 클라이언트 분기를 단순화 |
| pytest TestClient 의 startup 이벤트 충돌 | `with TestClient(app)` 컨텍스트가 `engine.begin()` 을 호출해 실DB 가 없는 테스트 환경에서 실패 | TestClient 를 컨텍스트 매니저 없이 생성해 startup 이벤트를 우회 |
| WHERE 절 검증의 false-positive | `is_public` 컬럼은 SELECT 절에도 등장해 단순 substring 검사가 실패함 | `WHERE` 절을 `split` 으로 잘라 해당 영역에서만 검증하도록 정밀화 |

## 9. 결론
개인 Prompt 관리 기능(`prompts` 테이블 + `/api/prompts` CRUD + Prompt 관리 SPA 뷰 + Chat 연동 + `💬 적용`) 이 요구사항대로 구현되었으며, 본인 / 타인 공개 / 타인 비공개 시나리오에서 권한 격리가 정확히 수행됨을 확인하였다. pytest 38건 신규 + 39건 회귀 = 총 77건 모두 통과하였고, 운영 환경(Docker)에 재빌드 및 force-recreate 적용 완료.
