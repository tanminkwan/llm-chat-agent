# Phase 2 요구사항 정의서 (Core Modules)

## 1. 개요
본 단계에서는 AI 에이전트의 핵심 기능인 LLM 호출 최적화, 대화 메모리 관리, 그리고 Qdrant 벡터 스토어 연동을 구현합니다.

## 2. 목표
- `chat-llm`과 `reasoning-llm`의 동적 전환 로직 구현
- 사용자/세션별 대화 이력 유지(Memory) 기능 구현
- Qdrant를 이용한 텍스트 임베딩 저장 및 유사도 검색 기능 구현

## 3. 상세 요구사항

### 3.1 LLM Gateway 고도화
- **Model Routing**: 단순 호출을 넘어, 시스템 프롬프트 주입 및 온도(Temperature) 등 파라미터 최적화.
- **Error Handling**: API 호출 실패 시 재시도(Retry) 로직 또는 폴백(Fallback) 메커니즘 검토.

### 3.2 Memory Manager (Conversation Memory)
- **Session Persistence**: 사용자의 세션 ID를 기반으로 과거 대화 내역을 조회하고 프롬프트에 포함.
- **Context Management**: 대화가 길어질 경우를 대비한 윈도우(Window) 기반 또는 요약(Summary) 기반 메모리 관리 검토.

### 3.3 Vector Store (Qdrant) 연동
- **Indexing**: 텍스트 데이터를 임베딩하여 Qdrant 컬렉션에 저장하는 기능.
- **Similarity Search**: 특정 텍스트와 유사한 문서를 검색하고 거리(Score)를 반환하는 기능.
- **RAG Preparation**: 검색된 문서를 LLM 프롬프트에 주입하기 위한 기초 유틸리티 구현.

## 4. 완료 조건 (Definition of Done)
- [ ] LLM Gateway 호출 시 세션별 메모리가 정상적으로 유지됨을 확인.
- [ ] Qdrant에 저장된 데이터가 검색 쿼리를 통해 정상적으로 반환됨을 확인.
- [ ] 모든 코어 로직에 대한 유닛 테스트 통과 (커버리지 95% 이상).
