# 시스템 상세 설계서 (Detailed Design Document)

## 1. 아키텍처 개요
본 시스템은 FastAPI 백엔드와 Vanilla JS 프론트엔드로 구성되며, LangChain을 통해 다양한 LLM 및 Vector Store(Qdrant)를 제어합니다.

## 2. 인증 및 권한 설계 (OIDC)
### 2.1 인증 플로우 (Authorization Code Flow)
1. **User → App**: `/login` 접근
2. **App → IDP**: `authorize_redirect` (Scope: openid, profile, email, groups)
3. **User → IDP**: 로그인 및 승인
4. **IDP → App**: `/auth/callback`으로 `code` 전달
5. **App → IDP**: `code`를 `Access/ID Token`으로 교환
6. **App → User**: 세션 쿠키 또는 JWT 발급 (현재 세션 기반 설계)

### 2.2 권한 매핑
- ID Token의 `groups` 클레임을 분석
- `Admin` 그룹 포함 시: 모든 API 접근 허용 및 관리자 메뉴 활성화
- `User` 그룹 포함 시: 일반 채팅 및 RAG 테스트 기능 허용

## 3. LLM Gateway 설계
### 3.1 인터페이스 (LLMGateway 클래스)
- `get_chat_llm()`: 일반 대화용 (gpt-4o 등)
- `get_reasoning_llm()`: 복잡한 추론용 (o4-mini 등)
- `get_embeddings()`: 텍스트 벡터화 (text-embedding-3-small 등)

### 3.2 모델 동적 전환
- Task의 특성에 따라 `reasoning-llm`을 호출할지 결정하는 라우팅 로직 포함

## 4. RAG 및 Vector Store 설계
### 4.1 데이터 스키마 (Qdrant Collection)
- `content`: 원문 텍스트
- `metadata`: 출처, 작성일, 사용자 ID 등
- `vector`: 임베딩 벡터 (1536 차원 권장)

### 4.2 거리 계산 로직
- Qdrant의 유클리드 거리(Euclidean) 또는 코사인 유사도(Cosine) 활용

## 5. 인프라 및 보안 설계
### 5.1 HTTPS 적용
- Port: `21443`
- Domain: `llm-agent.mwm.local`
- SSL: Self-signed certificate (개발용) 적용

### 5.2 환경 설정
- `.env` 파일을 통한 런타임 설정 주입 (Pydantic Settings 활용)
- `sample.env`를 통한 설정 가이드 제공

## 6. 모듈 구조 (Directory Structure)
- `apps/api/`: FastAPI 엔드포인트 및 서버 로직
- `libs/core/`: 인증(auth), 설정(settings), LLM 인터페이스(llm) 등 핵심 라이브러리
- `docs/`: 산출물 문서
- `certs/`: SSL 인증서
