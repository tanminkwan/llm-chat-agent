# Phase 05 요구사항 정의서 (Custom Embedding API Support)

## 1. 개요
프로덕션 환경에서 제공되는 단순화된 임베딩 API 규격을 수용하기 위해, 기존 OpenAI 규격 외에 커스텀 API 통신 방식을 선택적으로 활성화할 수 있는 기능을 구현함. 특정 모델명에 의존하지 않고 환경 변수 설정을 통해 범용적으로 대응함.

## 2. 목표
- 별도의 플래그를 통해 커스텀 임베딩 API 사용 여부를 결정함.
- 프로덕션 커스텀 API 규격(`{"tests": [...]}`)을 지원하는 범용 임베딩 구현체 도입.
- 기존 OpenAI 방식(default)과 프로덕션 커스텀 방식 간의 유연한 전환 지원.

## 3. 상세 요구사항

### 3.1 설정 및 분기 로직
- **환경 변수 추가**: `EMBEDDING_USE_CUSTOM` (boolean) 변수를 추가함.
- **기본값**: `false` (기존 `OpenAIEmbeddings` 방식 유지).
- **활성화**: `EMBEDDING_USE_CUSTOM=true`일 경우 프로덕션 커스텀 API 규격을 따르는 임베딩 클래스를 사용함.

### 3.2 커스텀 API 연동 규격
- **Request**: `POST` 요청, 본문 `{"tests": ["..."]}` 형식.
- **Response**: `{"embeddings": [[...], ...], ...}` 형식.
- **범용성**: `EMBEDDING_MODEL`에 설정된 모델명을 요청 시 함께 전달하거나 내부 로직에서 활용 가능하도록 함.

### 3.3 안정성 및 호환성
- LangChain `Embeddings` 인터페이스를 완벽히 준수하여 상위 서비스(RAGService 등)의 코드 변경을 최소화함.
- `httpx`를 이용한 동기/비동기 통신을 모두 지원함.

## 4. 완료 조건 (Definition of Done)
- [ ] `EMBEDDING_USE_CUSTOM` 변수가 `settings.py`에 정의됨.
- [ ] 플래그 설정에 따라 임베딩 인스턴스가 올바르게 생성됨을 유닛 테스트로 검증.
- [ ] 커스텀 API 통신 시 프로덕션 규격에 맞는 요청이 생성됨.
