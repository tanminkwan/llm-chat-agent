# Phase 05 요구사항 정의서 (Single Page Application 전환)

## 1. 개요
기존 다중 HTML 페이지(`/`, `/rag`, `/bulk`, `/admin`) 구조에서는 사이드바 메뉴를 클릭할 때마다 전체 페이지가 새로 로드되어 직전 화면의 작업 상태가 모두 초기화되는 문제가 있다. 본 단계에서는 4개 화면을 단일 SPA(Single Page Application) 구조로 통합하여, 페이지 이동 시에도 작업 상태가 보존되도록 개선한다.

## 2. 목표
- **상태 보존**: 채팅 대화 내용, 입력값, 검색 결과, 폼 상태 등 모든 화면 상태가 뷰 전환 후에도 유지됨.
- **무중단 네비게이션**: 사이드바 메뉴 클릭 시 서버 라운드트립 없이 즉시 화면이 전환됨.
- **기존 기능·디자인 유지**: 사용자 관점의 UI/UX 와 비즈니스 기능은 변경 없음. 백엔드 API 스펙도 그대로 유지.

## 3. 상세 요구사항

### 3.1 SPA 구조 전환
- **단일 진입점**: 모든 UI 라우트(`/`, `/rag`, `/bulk`, `/admin`)가 동일한 SPA 셸(`index.html`)을 반환함.
- **뷰 컨테이너**: 4개 화면(Chat / RAG / Bulk / Admin)은 모두 DOM 에 상시 존재하며, CSS `display` 토글로 표시·숨김을 제어함. 이를 통해 비활성 뷰의 입력값·스크롤·동적 생성 요소가 그대로 유지됨.
- **클라이언트 라우터**: History API(`pushState` / `popstate`) 기반의 경량 라우터를 자체 구현하여 URL 과 활성 뷰를 동기화함.

### 3.2 보존되어야 할 상태 (대표 시나리오)
- **Chat 뷰**:
  - 사용자/AI 메시지 누적 내역.
  - `thread_id` (페이지 로드 시 1회 생성, SPA 세션 동안 유지).
  - 입력 중인 메시지, 시스템 프롬프트, Temperature, 모델 선택값.
- **RAG 뷰**: 검색 조건(Collection / Domain / Algorithm / Query), 검색 결과 테이블.
- **Bulk 뷰**: Collection / Domain 선택값, 입력된 Source 값.
- **Admin 뷰**: 활성 탭(Collection / Domain), 폼 입력 상태, 목록 데이터.

### 3.3 라우팅 동작
- **초기 진입 / 새로고침**: `window.location.pathname` 을 해석해 해당 뷰를 활성화함. `/admin?tab=domain` 처럼 쿼리스트링도 해석.
- **메뉴 클릭**: 기본 브라우저 동작을 가로채(`preventDefault`) 뷰만 토글하고 `pushState` 로 URL 만 갱신함. `Ctrl/Cmd + Click`, 새 탭 등 사용자의 명시적 새 탭 동작은 그대로 허용.
- **브라우저 뒤로/앞으로 가기**: `popstate` 이벤트에서 URL 을 다시 해석해 뷰를 전환함.
- **외부 링크**: `📘 API Docs` (`/docs`, `target="_blank"`) 와 같은 외부/새 탭 링크는 라우터 가로채기 대상에서 제외.

### 3.4 권한 처리
- 기존과 동일하게 `/`, `/rag`, `/bulk` 는 인증된 사용자(User/Admin)만 접근. `/admin` 은 Admin 그룹만 접근 가능.
- 권한 체크는 SPA 셸을 반환하는 시점(서버 라우트 진입 시)에 수행.

### 3.5 사이드바 표시 규칙
- **Model Settings (Temperature)**: Chat 뷰에서만 노출. 다른 뷰로 이동하면 자동으로 숨김.
- **Admin Tools 패널**: Admin 그룹 사용자에게만 표시(기존 동작 유지).
- **활성 메뉴 강조**: 현재 활성 뷰에 해당하는 사이드바 링크에 강조 스타일 적용. Admin 뷰 내부에서는 활성 탭(`tab=collection|domain`)에 따라 하위 링크가 강조됨.

### 3.6 호환성 / 비기능 요구사항
- **백엔드 API 변경 금지**: `/chat`, `/api/*` 등 모든 REST 엔드포인트의 시그니처는 그대로 유지.
- **JS 모듈 분리 유지**: 기존 SRP 원칙(`chat.js`, `rag.js`, `bulk.js`, `admin.js`) 유지. SPA 라우터는 `app.js` 에서 단독 책임.
- **함수명 충돌 해소**: SPA 통합으로 인해 발생하는 전역 함수명 중복(`loadDropdowns` 등)을 제거.
- **점진적 폐기**: 기존 `rag.html`, `bulk.html`, `admin.html`, `partials/admin-modal.html` 은 더 이상 라우팅되지 않음(차후 정리).

## 4. 완료 조건 (Definition of Done)
- [ ] `/`, `/rag`, `/bulk`, `/admin` 가 모두 동일한 `index.html` 을 반환함.
- [ ] Chat 화면에서 대화 후 RAG 뷰를 다녀와도 메시지 내역과 입력값이 그대로 남아 있음.
- [ ] 사이드바 메뉴 클릭 시 페이지 새로고침이 발생하지 않음(개발자 도구 Network 탭으로 확인).
- [ ] 브라우저 뒤로/앞으로 가기 동작이 SPA 뷰 전환과 정확히 동기화됨.
- [ ] `/admin` 에 비-Admin 계정으로 접근 시 `unauthorized` 페이지로 리다이렉트됨.
- [ ] 기존 단일 페이지에서 동작하던 모든 기능(채팅 스트리밍, RAG CRUD, 엑셀 업로드/삭제, Collection·Domain 관리)이 SPA 환경에서도 동일하게 동작함.
