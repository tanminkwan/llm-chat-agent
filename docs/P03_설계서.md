# Phase 3 상세 설계서 (Data & RAG)

## 1. Vector Store (Qdrant) 아키텍처

### 1.1 Qdrant 클라이언트 설계 (`libs/core/vectorstore.py`)
- **Library**: `qdrant-client` 사용.
- **Connection**: `Settings`에 정의된 `QDRANT_URL`을 통해 비동기 클라이언트 유지.
- **Error Strategy**: 서버 미기동 시 에러 핸들링 및 재연결 로직.

### 1.2 컬렉션 및 인덱스 설계
- **Name**: `llm_agent_docs`
- **Vector Params**:
  - `size`: 1536
  - `distance`: `Cosine`
- **Payload Indexing**: 메타데이터 검색 속도 향상을 위해 `filename`, `user_id` 등에 대한 페이로드 인덱스 생성.

## 2. RAG 파이프라인 설계

### 2.1 Retrieval 흐름
1. User Query 수신
2. Query Embedding 생성 (OpenAI `text-embedding-3-small`)
3. Qdrant `search` API 호출 (Top-K=3)
4. 결과 텍스트 결합 (Context String 생성)

### 2.2 Prompt Engineering
- **Template**:
  ```text
  당신은 아래 제공된 지식을 바탕으로 질문에 답하는 어시스턴트입니다.
  모르는 내용이라면 지어내지 말고 모른다고 답하세요.
  
  [지식 베이스]
  {context}
  
  질문: {question}
  ```

## 3. 구현 단계 (TDD)
