# P07 설계서 - 대화 쓰레드(Thread) 관리 및 로깅 개선

## 1. 변경 대조 (AS-IS vs TO-BE)

| 구분 | AS-IS (기존) | TO-BE (변경 후) |
| :--- | :--- | :--- |
| **용어/필드명** | `session_id` | **`thread_id`** |
| **기억의 단위** | `user_{sub}` (사용자별 고정) | **Thread ID** (페이지 로드 시마다 갱신) |
| **로깅 필드** | `session_id` (응답로그엔 누락) | **`thread_id` (요청/응답 모두 포함)** |

## 2. 세부 설계

### 2.1 스키마 수정 (apps/api/schemas.py)
- `ChatRequest` 클래스의 `session_id` 필드명을 `thread_id`로 변경.

### 2.2 백엔드 수정 (apps/api/main.py)
- `chat` 함수 내에서 `request.thread_id`를 참조하도록 수정.
- `request_log` 및 `response_log`의 키값을 `thread_id`로 변경.

- `const threadId = crypto.randomUUID()` 사용.
- API 호출 시 `thread_id: threadId`로 전송.
- **에러 핸들링**: 스트리밍 수신부에서 `data.content`가 없더라도 `data` 객체 전체를 분석하여 에러 메시지 등을 사용자에게 표시하도록 개선.

## 3. 로깅 데이터 규격 (예시)

### 3.1 Request Log (JSON)
```json
{
  "request_id": "req_123",
  "thread_id": "thread_uuid_001",
  "type": "request",
  "messages": [...]
}
```

### 3.2 Response Log (JSON)
```json
{
  "request_id": "req_123",
  "thread_id": "thread_uuid_001",
  "type": "response",
  "full_response": "..."
}
```

### 3.3 Error Log (JSON)
```json
{
  "request_id": "req_123",
  "thread_id": "thread_uuid_001",
  "type": "error",
  "error": "OpenAI API Key quota exceeded"
}
```

## 4. 검증 계획
1. 브라우저 새로고침 후 첫 메시지 전달 시 새로운 `thread_id`가 생성되는지 확인.
2. 백엔드 로그(stdout)에서 질문, 답변, **에러 로그** 모두에 동일한 `thread_id` 키값이 기록되는지 확인.
