# P06: 오프라인용 REST API 문서(Swagger) 구축 설계서

## 1. 아키텍처 개요
FastAPI의 기본 Swagger UI 생성 로직을 오버라이딩하여, 정적 리소스를 외부 CDN이 아닌 로컬 경로(`/static/swagger/`)에서 불러오도록 설계함.

## 2. 세부 설계 내역

### 2.1 정적 리소스 구조
- **경로**: `apps/api/static/swagger/`
- **필수 파일**:
  - `swagger-ui.css`: 스타일 시트
  - `swagger-ui-bundle.js`: 메인 로직
  - `swagger-ui-standalone-preset.js`: 독립 실행용 프리셋
  - `favicon.png`: 아이콘

### 2.2 백엔드 수정 (apps/api/main.py)
- **FastAPI 초기화 변경**: `docs_url=None`, `redoc_url=None` 설정을 통해 기본 자동 생성 엔드포인트 비활성화.
- **커스텀 라우트 구현**:
  - `GET /docs`: `get_swagger_ui_html()`을 호출하며, `swagger_js_url` 및 `swagger_css_url` 인자에 로컬 경로 지정.
  - `GET /redoc`: `get_redoc_html()`을 호출하며 로컬 경로 지정.
  - `GET /openapi.json`: 기본 OpenAPI 스펙은 그대로 유지.

### 2.3 프론트엔드 수정 (Sidebar 통합)
- **대상**: `index.html`, `rag.html`, `bulk.html`, `admin.html`
- **변경**: `nav` 태그 내 `Batch Upload` 링크 하단에 `API Docs` 링크 추가.
- **코드 예시**:
  ```html
  <a href="/docs" target="_blank" class="nav-link" style="...">📘 API Docs</a>
  ```

## 3. 검증 계획
- **오프라인 시뮬레이션**: 브라우저 개발자 도구의 Network 탭에서 `cdn.jsdelivr.net` 등 외부 도메인 호출이 발생하는지 확인. 모든 리소스가 로컬 도메인(`llm-agent.mwm.local` 등)에서 응답해야 함.
- **동작 확인**: `/docs` 접속 시 API 목록이 정상적으로 출력되고, 'Try it out' 기능을 통해 실제 API 호출이 가능한지 확인.
