# P06: 오프라인용 REST API 문서(Swagger) 구축 요구사항

## 1. 개요
- 운영 환경(Air-gapped)에서 인터넷 연결 없이도 REST API 명세서를 확인할 수 있도록 Swagger UI를 로컬화하여 구축함.
- 프로젝트 골든 룰(Rule 5.26: No External CDNs)을 준수함.

## 2. 목표
- 외부 CDN 참조 없이 서버 내부 리소스로만 동작하는 Swagger UI 제공.
- 관리자 및 사용자가 손쉽게 API 명세에 접근할 수 있도록 UI 메뉴 연동.

## 3. 세부 요구사항
### 3.1 Swagger UI 로컬화
- **정적 리소스 포함**: `swagger-ui.css`, `swagger-ui-bundle.js` 등 필수 정적 파일을 서버의 `/static/swagger` 폴더 내에 포함함.
- **CDN 의존성 제거**: FastAPI의 기본 `/docs`, `/redoc` 엔드포인트가 외부 CDN을 호출하지 않도록 커스텀 엔드포인트로 대체함.

### 3.2 UI 메뉴 연동
- **사이드바 통합**: 모든 메인 HTML 페이지(`index.html`, `rag.html`, `bulk.html`, `admin.html`)의 사이드바 내비게이션에 "📘 API Docs" 메뉴를 추가함.
- **접근 권한**: API 문서는 기본적으로 모든 인증된 사용자에게 오픈하되, 운영 정책에 따라 관리자로 제한할 수 있는 구조를 유지함. (현재는 전체 인증 사용자 대상)

## 4. 제약 사항
- 운영 환경은 인터넷이 차단된 상태이므로, 정적 파일을 다운로드하여 소스 코드에 포함시켜야 함.
- 파일 단 한 줄의 수정이라도 `docker build --no-cache`를 통해 반영함.
