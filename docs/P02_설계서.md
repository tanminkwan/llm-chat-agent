# Phase 2 상세 설계서 (Core Modules)

## 1. LLM Gateway & 권한 연동 설계

### 1.1 권한별 시스템 프롬프트 (System Prompts)
- **Admin**: "당신은 시스템 관리자 권한을 가진 AI 에이전트입니다. 모든 시스템 로그 및 데이터에 접근할 수 있으며, 복잡한 추론 작업을 수행합니다."
- **User**: "당신은 일반 사용자용 AI 에이전트입니다. 친절하고 정확하게 정보를 제공합니다."

### 1.2 모델 라우팅 전략
- `LLMGateway`는 요청자의 `UserInfo.is_admin` 값을 확인하여 기본 모델을 선택하거나 추가 기능을 활성화함.

## 2. Memory Manager 설계 (대화 유지)

### 2.1 세션 기반 메모리 (`libs/core/memory.py`)
- **구현**: LangChain의 `InMemoryChatMessageHistory`를 확장.
- **구조**: `dict[session_id, ChatMessageHistory]` 형태로 서버 메모리에 대화 이력 저장.
- **생명주기**: 서버 재시작 전까지 유지 (향후 Redis 등으로 확장 가능하도록 인터페이스 분리).

## 3. Vector Store (Qdrant) 연동 설계

### 3.1 Qdrant 스키마
- **Collection Name**: `llm_agent_docs`
- **Vector Size**: 1536 (OpenAI `text-embedding-3-small` 기준)
- **Distance Metric**: `Cosine`

### 3.2 문서 인덱싱 및 검색 흐름
1. `libs/core/vectorstore.py`가 싱글톤(Singleton)으로 Qdrant 클라이언트 유지.
2. `Admin` 사용자가 문서를 업로드하면 임베딩 후 저장.
3. 모든 질문 시 관련 컨텍스트를 검색하여 프롬프트에 자동 주입 (RAG).

## 4. TDD 및 검증 계획
- **Memory Test**: 여러 세션이 섞이지 않고 각자의 대화 내용을 기억하는지 테스트.
- **Vector Search Test**: 특정 키워드로 검색 시 관련 문서가 상위권에 나오는지 테스트.
- **Integrated Test**: 인증된 사용자의 요청이 `Memory -> VectorStore -> LLM`을 거쳐 응답되는지 전체 흐름 검증.
